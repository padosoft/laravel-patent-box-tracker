# Evidence Strategy

The tracker becomes stronger when the repository already contains durable project evidence.

::: grids
::: grid
::: card
**Design docs**

Keep ADR, plan, spec, and lesson files close to implementation.
:::
::: card
**Commit discipline**

Prefer smaller commits with messages that state intent and technical outcome.
:::
::: card
**Fiscal boundaries**

Run sessions per fiscal year and avoid mixing unrelated IP outputs.
:::
:::
:::

## Recommended Files

- `PLAN.md` for scoped implementation plans,
- `ADR-0001-name.md` for decisions,
- `LESSON.md` for lessons learned,
- release notes for user-visible IP increments,
- prompt and model records when AI materially contributed.
